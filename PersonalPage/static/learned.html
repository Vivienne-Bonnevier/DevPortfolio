Syntax

An important part of proper HTML writing is proper syntax. While many browsers can sometimes compensate for errors in HTML, the best practice is to ensure that syntax is done correctly. It also assists with programs like screen readers and crawlers to have proper syntax. For example, knowing how to properly use html elements like headers, footers, articles, and lists can assist machines that don't rely on eyes like we do.

Layouts

While knowing HTML is important, it's also important to be able to apply proper CSS. CSS allows HTML to be more easily read in ways that make sense for human brains. It also allows displayed information to be more stylish and engaging for users. Useage of proper layout, like flexboxes, allows for information to be fit in interesting and engaging ways.

Accessibility

Web-developers must also ensure that their pages are accessible to anyone. This means accomodating disabilities by making sure that their pages work well with screen readers (by use of proper syntax, alt-text on images, and good design practices). It is also good practice to accomodate motor disabilities as well (for example: by incorporating tab navigation).

Responsiveness

Since there are a wide variety of devices used by consumers, webpages should try to be useable on as many as these as possible. This means changing page layouts for different screen sizes. Sometimes it may mean accomodating different types of browsers or offering compatibility with older devices as well.

Style

No matter if a page is commercial or for a non-profit, in our world it is important to stand out and make a good impression on consumers. By using good style practices including appealing colors designs, fonts, and more, webpages can stand out and serve to be pleasant experiences. For example: using interesting fonts from the google api can help certain text be more visually appealing!

Animations

Human eyes are drawn to movement, and as such making use of animations and transition effects can help to liven up a web page and draw attention to important details. By making use of mouse hovers, web-develoeprs can also use animation and transitions to fit more content on a page by having that information shrink when not hovered over and expand when it is.
